1.1 The Basics of the Web and HTML

What is a programmer?

A programmer is a person that tells computers what to do. Computers are excruciatingly specific, they care a lot about syntax and structure and following rules. The job of the programmer is to break down things they want to happen into constituent parts.

How does a programmer do this?

They use at least five ways of thinking. The ways of thinking include procedural thinking, abstract thinking, systematic debugging, technological empathy and systems thinking.

As a programmer, you do not need to remember every single detail. Google and Stack Overflow exist for that (What is Stack Overflow for? Check out this FAQ page). However, you do need to understand the big concepts so that you can wrap your head around problems.

Knowing where to look for the answer is half of the solution. Also, get used to reading documentation. Proper documentation is the secret source to understanding how any (properly documented!) system works.

Almost anyone can learn almost anything given enough time and resources.

Specifically

  • Tags must be opened and closed, unless they are void (containing no content, e.g. br)
  • An element is a pair of tags (opening and closing)
  • Tags can either be inline or block (effectively create an invisible box around the content)

1.2 Creating a Structured Document

Broadly

  • Building with HTML, CSS and JS is a little bit like building a house. HTML lays out the structure, ie where the rooms are. CSS sets the style, such as what colour the walls are. Javascript provides the interaction, like the garage door that opens and closes when you press a button.
  • The web is made up of boxes. Start your designs with large boxes, and work your way down to the detail.
  • Crossing the knowledge gap requires reflection. Crossing the skills gap requires practice.

What is the DOM?

The DOM is how objects in a web page are represented. It sorts out how all of the little components, like images, text, links and titles are shown.

The Tree Like Structure of HTML

Earlier, we talked about how the web is made of boxes. These boxes, or elements, live inside each other. This is why we say that HTML has a nested, or tree-like structure.

Other Things I Learned...

  • DOM = Document Object Model
  • Tags are turned into elements in a tree, as part of the DOM
  • A circle is just a square with a border radius of 50%
  • Tags either contain nothing, text or another element. This is true of every single website you visit. That's (1) kind of crazy, but (2) very reasuring. Everything you have even seen in the browser is at some level composed of the same units and rules that you're using right now.

1.3 Adding CSS Style and HTML Structure

DRY

Do not Repeat Yourself. You're learning to be a programmer. The machine is there to do mechanical, repetitive things for you, to make your life easier.

Avoiding repetition is kind of magical. It makes things consistent, and when you realise something is wrong, this helps you make changes in less time.

Code, Test, Refine

Are things not quite like you'd like them to be? There's a couple of questions you should ask:

  1. Is the structure of the document correct? Divide and conquer. Look for places that are distinct, natural boxes for content.
  2. Are the boxes the right size?
  3. Are the boxes in the right position?

Specifically

  • Use a logical tag. Don't create a div and create a title class, just use h1. There are conventions in place. Read about them and make sure your code adheres to best practice. Not only will it make it clearer, it will make it easier for other people to pick up what you have been working on.
